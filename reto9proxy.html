    Documentacion de la implementacion de un servidor Proxy

    En este apartado de la pagina os enseñaremos la documentacion del proceso de implementacion de Servidor Proxy

    Paso 1: Actualizaremos los paquetes del sistema Ubuntu Server

    Antes de empezar a instalar cualquier tipo de paquetes en Ubuntu Server debemos asegurarnos de que todos los paquetes del sistema esten actualizados para asegurar que todo funciona correctamente. Para ello ejecutaremos el comando:

    sudo apt-get update

    Paso 2: Instalaremos el paquete de Squid

    Una vez actualizados todos los archivos del sistema operativo, ejecutaremos este comando para instalar los paquetes relacionados con Squid en Ubuntu Server

    sudo apt-get -y install Squid

    Este comando instalará el paquete del servidor Squid en el sistema.

    Paso 3: Automatiza el Paquete y sus funciones.

    Una vez que se complete la instalación, debemos escribir el comando systemctl enable Squid.

    Este comando permite a Squid el poder funcionar automaticamente cada vez que el equipo se encienda, haciendolo mas practico al no necesitar de activar se cada vez que el equipo se encienda

    Paso 4: Configura el archivo /etc/Squid/whitelist

    En el archivo /etc/Squid/whitelist, pondremos nuestra IP, y habremos de recargar el servicio con el comando:

    curl ifconfig.me >> /etc/squid/whitelist service squid restart

    Este comando reiniciará el servicio Squid en el sistema.

    Paso 5: Descargar paquetes para uso contraseñas de usuario Proxy

    Descargaremos el paquete de apache2-utils para configurar las contraseñas de usuario en el servidor proxy

    apt-get install apache2-utils -y

    Paso 6: Configuracion de miembros de Squid

    El paquete de Utility preguntara por una contraseña, la crearemos y nos tendremos que acordar.

    Paso 7: Integrar mecanismo de seguridad con Squid

    Integramos un mecanismo de seguridad con Squid Daemon, edita el archivo config con los comandos:

    auth_param basic program /usr/lib/squid3/basic_ncsa_auth /etc/squid/pswds auth_param basic realm proxy acl authenticated proxy_auth REQUIRED http_access allow authenticated

    Paso 8: Prueba final

    Para comprobar de que la instalacion ha ido bien escribiremos el siguiente comando:

    curl -v -x http://nombre de usuario:contraseña Direccion IP:puerto https://google.com

    Conclusión

    En esta documentacion, te hemos mostrado cómo nuestro grupo ha conseguido instalar el servicio Squid y como configurar lo correctamente en un servidor Ubuntu. Ahora puedes usar el servicio Squid según tus necesidades.
